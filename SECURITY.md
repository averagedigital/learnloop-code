# Политика безопасности

## Поддерживаемые версии

CodeLearnML находится в статусе beta. Исправления безопасности выпускаются только для последней версии `0.0.x` в ветке `main`.

| Версия | Поддержка |
| --- | --- |
| 0.0.x | Да |
| Более ранние версии | Нет |

## Сообщение об уязвимости

Не публикуйте сведения об уязвимости в открытых Issues или Discussions. Используйте [приватное сообщение об уязвимости GitHub](https://github.com/averagedigital/learnloop-code/security/advisories/new) и укажите:

- затронутую версию и компонент;
- шаги воспроизведения или минимальный proof of concept;
- ожидаемое влияние;
- предлагаемый способ устранения, если он известен.

Не прикладывайте реальные API-ключи, пользовательские данные и другие секреты. Мы проверим сообщение и согласуем раскрытие информации после подготовки исправления.

## Граница выполнения кода

CodeLearnML выполняет недоверенный код задач только через Judge0. Локальные server и worker Judge0 запускаются в privileged Docker-контейнерах, необходимых его sandbox runtime. Оставляйте API Judge0 привязанным к `127.0.0.1`, не открывайте его в недоверенную сеть и перед развёртыванием изучите [политику безопасности Judge0](https://github.com/judge0/judge0/security/policy).

Beta-версия предназначена для локальной оценки и не является защищённым многопользовательским production-сервисом.
